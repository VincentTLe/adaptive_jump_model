Từ nguồn dữ liệu tới bằng chứng có hash

Downloader nghiên cứu không chỉ tải một bảng số. Nó phải chứng minh đã dùng đúng contract, đúng nguồn, đúng thời gian và không âm thầm sửa dữ liệu.

Phạm vi: acquisition engineering, chưa có return, feature, state hay backtest.

1. Hãy nghĩ như nhận một phong bì niêm phong

Raw payload

Thứ nguồn gửi về

HTTP response của FRED/BOJ, hoặc bảng chưa biến đổi do yfinance trả về. Đây là bằng chứng gần nguồn nhất.

Canonical observations

Thẻ mục lục tối giản

Chỉ còn hai cột date,value để code sau đọc nhất quán. Không fill, không tính return, không sửa outlier.

Ví dụ giả lập, không phải dữ liệu dự án 
Raw source                    Canonical
observation_date,RATE         date,value
2020-01-02,1.50        →      2020-01-02,1.50
2020-01-03,.            →      2020-01-03,

Dấu chấm nghĩa là thiếu. Canonical giữ ô trống; nó không đoán lãi suất ngày đó bằng ngày hôm trước.

4. SHA-256 trả lời câu hỏi nào?

hash = SHA256(chính xác các bytes trong file)

Hai file giống từng byte cho cùng hash. Chỉ một ký tự đổi sẽ tạo hash khác. Manifest lưu hash raw và canonical để lần sau biết dữ liệu có thật sự giống không.

Hash không chứng minh dữ liệu đúng về kinh tế. Nó chỉ chứng minh file đang kiểm tra đúng là file đã được ghi nhận. Định nghĩa source và proxy vẫn phải audit riêng.

Artifact được tạo tại data/raw/<run_id>/manifest.json. Receipt độc lập nằm tại artifacts/data-acquisition/<run_id>/validation.json. Cả hai đều ignored, không commit raw market data.

7. Bước này chứng minh và không chứng minh gì?

Đã chứng minh
  • CLI đọc đúng contract.
  • Sáu source đi qua validation chung.
  • Files, hashes, quality facts và cutoff khớp.
  • Run gắn với code SHA sạch.
Chưa chứng minh
  • Proxy giống dữ liệu Bloomberg/GFD.
  • Monthly rate được căn ngày đúng.
  • Feature hay model đúng paper.
  • Chiến lược có Sharpe tốt hơn.

Tự kiểm tra

Tại sao không forward-fill lãi suất ngay ở downloader?

Forward-fill cần một giả định về ngày thông tin trở nên biết được. Đó là protocol causal, không phải acquisition.

Run có raw files nhưng không có manifest có hợp lệ không?

Không. Manifest chỉ được ghi cuối cùng; thiếu nó nghĩa là run đã dừng giữa chừng hoặc chưa qua đủ sáu source.

Git SHA có đủ khi source code đang dirty không?

Không. Vì output sẽ đến từ code khác với commit đó. Runner phải reject trước network.

Acquisition pass có phải replication pass không?

Không. Đây chỉ là ENGINEERING / SMOKE. Replication cần features, walk-forward models, backtest và gate riêng.
