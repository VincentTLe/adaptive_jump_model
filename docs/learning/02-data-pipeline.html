Question 2

Can another person obtain the same input?

Naming a website is not enough. A research run needs a record of the exact bytes it used.

Remember this: reproducible data has an identity, a time boundary, and a receipt.

A tiny receipt

Imagine two files named prices.csv. One has 100 rows and the other has 101. Their names match, but the experiments do not.

source: example.org/prices
downloaded: 2026-07-12
last allowed date: 2023-12-31
rows: 100
sha256: 7f1a...c902

The long SHA-256 hash is a fingerprint calculated from every byte. Change one number and the fingerprint changes.

The current pipeline in five steps

1. ContractDefine series and dates
2. FetchDownload raw observations
3. ValidateReject duplicates and bad rows
4. CutRemove dates after 2023
5. SealWrite hashes and manifest

A contract is the written rule for what is allowed. A cutoff is the latest permitted date. A manifest is the machine-readable receipt listing the source, row count, dates, definition, and hash.

Why the cutoff is enforced before modeling

Allowed

Download a source, then mechanically remove every observation after 31 December 2023 before any feature or model can read it.

Not allowed

Keep later rows in a convenient file and merely promise that the model will ignore them.

The second approach makes accidental future access too easy. The current project separates the permitted dataset from later extension evidence.

What this receipt proves

It can prove

Which file was used, where it came from, its stated definition, its date range, and whether it changed.

It cannot prove

That a free proxy is economically identical to Bloomberg/GFD data or that the model is scientifically correct.

Key distinction: reproducibility means rerunning the same defined experiment. Data parity means matching the paper's economic input. We need both for exact replication.
Optional deeper look: why a hash is useful but limited

A hash is deterministic: the same bytes produce the same value. It detects accidental changes, but it does not judge whether the contents are correct. A perfectly hashed price index is still not a total-return index.

Check your understanding

Question: A file has a valid hash but contains dates from 2024. Is it allowed in the frozen replication?
