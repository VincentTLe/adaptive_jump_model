Question 4

Did the Jump Model actually win?

The experiment compares three ways to choose between the market and cash.

Remember this: a complicated model earns scientific attention only if it beats simple alternatives under the same rules and dates.

A two-choice strategy

Suppose you have $100 tonight. The model does not predict tomorrow's exact price. It labels the market as a favorable or unfavorable regime, meaning a period with a broadly similar pattern.

Favorable state$100 in market
Unfavorable state$100 in cash

The three competitors

Buy and hold

Always stay in the market. No regime model is used. This is the simplest baseline.

Hidden Markov Model

Assumes returns come from hidden states that switch with learned probabilities. “Hidden” means the state is inferred, not directly observed.

Jump Model

Groups the three features into states and charges a penalty when the state changes. A larger fixed penalty favors more persistent regimes.

How settings are chosen without using the test

FitLearn from trailing history
ValidateReplay eight past years online

TestRecord the untouched next result

A hyperparameter is a model setting chosen outside the fitting formula. Here it includes the Jump Model penalty and the HMM state filter. Selection uses past validation Sharpe only.

What “win” means

Sharpe ratio compares average excess return with the variability of strategy returns; higher is better. maximum drawdown is the largest peak-to-trough wealth loss; a smaller loss is better.

Directional gate for each market: the Jump Model must have higher Sharpe than both HMM and buy-and-hold, and a smaller maximum drawdown than buy-and-hold. The same preregistered primary delay is used everywhere.
Current frozen conclusion: proxy non-replication. The verified v7 run passed its engineering and protocol checks, but zero of three markets passed the full directional gate. Therefore adaptive-model work remains blocked. This does not establish that the paper is wrong because the project used free proxy data rather than the paper's Bloomberg/GFD inputs.

Optional deeper look: fixed Jump Model objective

The model balances two quantities: feature observations should stay close to their assigned state center, while state changes add a fixed cost called lambda. Dynamic programming finds the lowest total-cost state path. The backtest uses only the terminal state available on each date.

Answer: No. The frozen gate requires all three comparisons, not whichever metric looks favorable afterward.
