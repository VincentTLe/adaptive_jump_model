Part III · Features And Models

From Returns To Model Features

How can a noisy list of daily gains and losses become three stable measurements that a model can compare?

Beginner levelDeterministic feature laboratoryPaper Section 3.4.1 and Table 2
The opening question

Should one bad day erase everything we knew yesterday?

You invest $100 in an equity index while cash earns nothing in this tiny example. The index loses 4% today, so the account becomes $96. The daily excess return is therefore -4%. That number says today was bad, but it does not answer three broader questions: Was this one isolated loss or part of recent instability? Are recent gains large enough to compensate for recent downside? Does the short view tell the same story as the slower view?

Raw observation

$100 → $96

The -4% excess return describes one interval. Tomorrow it will be replaced by another noisy observation.

A common failure: one-day diagnosis

One loss ≠ a regime

Calling every negative day a bear state creates rapid switches, costs, and false alarms. A feature should summarize recent history without seeing the future.

A model cannot reason directly about your anxiety or about a story in the news. It receives numbers. A feature is a deliberately constructed number that exposes a useful aspect of the observations. In this project, three features summarize recent excess returns: downside deviation over a fast memory, and two Sortino ratios over medium and slow memories.

The transformation does not predict tomorrow and does not prove a regime exists. It changes the representation. Instead of asking a model to discover every idea from one jagged return series, we provide one measure of recent harmful variation and two measures of reward relative to harmful variation. Every value at date t uses only returns through date t.

The idea to rememberA feature is a causal summary, not new evidence. Exponential memory gives recent observations more influence; downside deviation measures harmful variation; the Sortino ratio compares average excess return with that downside. Standardization then places the three columns on comparable numerical scales.
Words before notation

Build a precise vocabulary for memory and scale

Excess return

Equity simple return minus the causally aligned cash return for the same observation. It measures reward above the available safer alternative.

Exponential weight

A positive historical weight that shrinks by a constant factor as an observation gets older. Recent rows count more, but older rows do not abruptly disappear.

Half-life

The age, in observations, at which an unnormalized exponential weight is half the weight of the newest observation. A 10-day half-life describes memory, not a ten-row cutoff.

Downside deviation

The square root of a weighted average of squared negative excess returns. Positive returns contribute zero to its loss term.

Sortino ratio

A weighted average excess return divided by downside deviation computed at the same half-life. It is unitless when numerator and denominator use the same return unit.

Standard score

A value after subtracting a training-window mean and dividing by that window's population standard deviation. It says how unusual the feature is relative to permitted history.

Deep explanation: memory, downside, missing values, and scale

Simple return is the percentage change in value over one interval. A 1% return is stored as the decimal 0.01. Cash return is the daily return of the local safer asset proxy. Excess return is not the equity return alone: if equity earns 0.04% and cash earns 0.01%, excess return is 0.03%, or 0.0003 as a decimal.

An exponentially weighted moving, or EWM, statistic uses all available earlier observations with decaying influence. The word moving means the endpoint advances one date at a time. The word weighted means observations do not count equally. A half-life of 20 gives a return from 20 observations ago half the raw weight of today's return; at age 40 it receives one quarter.

The project uses adjusted EWM weights. At each endpoint, the finite historical weights are normalized to sum to one. Early in a series, this avoids pretending that an infinite prehistory of zero-valued observations existed. The project also preserves missing observations instead of silently filling them. A missing excess return makes the feature on that row missing, and model input later requires complete feature rows.

Downside means returns below zero in this protocol. It is not the same as volatility. A +5% surprise increases ordinary squared variation, but it contributes zero to the downside square. Squaring prevents negative and positive signs from canceling within the loss measure and gives larger losses more influence. Taking the square root restores the original return unit.

The word Sortino names a reward-to-downside measure. A positive numerator and small denominator produce a positive, larger ratio. A negative recent mean produces a negative ratio. If every weighted return is nonnegative, downside deviation is zero and division is undefined; the project records the Sortino feature as missing rather than inventing infinity or adding an undocumented constant.

Standardization is distinct from annualization. Annualization changes a daily statistic into a yearly convention, often using 252 trading days. Standardization changes coordinates to center zero and scale one within a training window. The frozen model features are not annualized. Their rolling training-window scaler uses three separate means and three separate population standard deviations, with zero degrees-of-freedom correction, as implemented by scikit-learn's StandardScaler.

Worked example two

Follow one loss through the three feature columns

Consider eight invented excess returns: +1%, -1%, +0.5%, -4%, +1%, -0.5%, +1.5%, and -1%. With a teaching half-life of 10, normalized adjusted weights rise gradually from about 9.7% for the oldest row to about 15.7% for the newest row. The weighted mean is about -0.31%.

Only four negative rows enter the downside sum. Each is squared, multiplied by its endpoint weight, and added. The square root gives a downside deviation of about 1.48%. Dividing the -0.31% weighted mean by 1.48% gives a teaching Sortino ratio near -0.21. The ratio is negative because weighted reward is negative, not because downside deviation itself is negative; a deviation cannot be below zero.

Now imagine the -4% observation were +4%. The downside feature would fall because that row would contribute zero instead of a squared loss. The weighted mean would rise. Both changes would push the Sortino ratio upward. This is why the three features are related but not duplicates: downside records one risk dimension, while Sortino combines direction and downside at a chosen memory speed.

A common failure: mixing half-lives inside one ratio

Suppose someone divides a 20-day EWM mean by the 10-day downside feature because DD-10 is already available. That creates a different statistic from the paper's Sortino-20. The denominator for Sortino-20 must be downside deviation computed with half-life 20; Sortino-60 uses both a 60-day mean and a 60-day downside denominator. The standalone risk feature remains DD-10.

Another common failure fits one scaler over the entire 1970-2023 sample. A late crash then changes standardized features assigned to earlier years. The correct sequence is: construct causal unscaled features, take the current trailing 3,000 complete model rows, fit the three scaler parameters on that window only, and transform that window for the current model fit or terminal inference.

A shorter half-life concentrates more weight near Day 8, so the older Day 4 shock matters less relative to the newest rows. A longer half-life makes weights more even and preserves the shock's influence. This does not mean one half-life is universally superior. The paper preregisters different speeds because risk may change quickly while average return is noisier and benefits from longer memory.

Connection to the named paper

Table 2 fixes a small, shared feature set

Section 3.4.1 of Shu, Yu, and Mulvey (2024), arXiv:2402.05272v3 defines three JM features from an excess-return series. Table 2 lists EWM downside deviation with half-life 10 and EWM Sortino ratios with half-lives 20 and 60. The same feature design is used for the S&P 500, DAX, and Nikkei 225 rather than customizing features after inspecting each market.

The authors motivate downside deviation by investors' concern with harmful losses rather than upside uncertainty. They use a shorter memory for the smoother risk measure and two longer memories for noisier return information. The two Sortino features give return two views while retaining one direct risk feature. Section 3.4.1 also says the parsimonious set supports a fair comparison with the one-dimensional return input used by the HMM.

Figure 3 plots fitted JM feature centroids by regime over rolling windows. Its displayed values are transformed for interpretation, including annualization and plotting rescalings described in the caption. Those display choices must not be confused with the raw model feature contract. Chapter 13 will read the figure in context; this chapter establishes only how the model inputs are constructed.

The paper mentions broader feature sets, sparse feature selection, macroeconomic indicators, and cross-asset variables as possible research directions. Those are proposals, not part of the replication protocol. Adding them now would change the scientific question and increase researcher degrees of freedom.

Connection to this project

The code preserves the feature contract explicitly

src/adaptive_jump/features.py first computes simple and log equity returns between consecutive valid index levels. It causally aligns the local cash yield, converts that annual percentage proxy to a daily decimal under the frozen convention, and defines excess_return = equity_simple - cash_return.

The make_features function clips excess return only from above at zero to form the downside term; this is not outlier clipping. It squares that nonpositive series, computes adjusted EWM means, and takes a square root. Each Sortino numerator and denominator use the same requested half-life. Infinite ratios are replaced by missing values, and any row with missing excess return keeps all feature outputs missing.

negative_squared = excess.clip(upper=0).pow(2)
downside = sqrt(EWM(negative_squared, halflife=h).mean())
sortino_h = EWM(excess, halflife=h).mean() / downside_h

The frozen choices live in research.toml: half-lives 10, 20, and 60; adjusted EWM; no burn-in; no feature annualization; no outlier clipping; and zero downside recorded as missing. src/adaptive_jump/models.py fits one StandardScaler on each trailing 3,000-row JM window and records its means and scales in refit diagnostics.

tests/test_features.py checks formulas against direct pandas calculations, missing behavior, cash availability, and future-prefix invariance. Model tests check that scaling is fitted on the trailing window and recorded. These tests establish implementation consistency. They do not establish that these three features are the best economic representation.

The decay equation guarantees the half-life interpretation. An observation aged h has raw weight qhh=2−1=1/2. The denominator normalizes all weights that actually exist through endpoint t. There is no term from t+1, so the construction is causal.

For a positive return, min(Ri,0)=0. For a -4% return stored as -0.04, its squared downside is 0.0016. Weighted squares are averaged before the root; averaging already-rooted absolute losses would define a different statistic.

After constructing the three unscaled columns, feature j in the current training window is standardized as zt,j=(xt,j−μj)/σj. Here μj and σj come only from that trailing training window. Standardization prevents a numerically large DD coordinate from automatically dominating a smaller Sortino coordinate in Euclidean distance.

Evidence and limits

Separate verified arithmetic from economic belief

What is verified

The locked PDF names the three features. Config, source, and tests agree on return alignment, half-lives, EWM options, missing values, trailing scaling, and absence of feature clipping or annualization.

What remains empirical

Whether these features separate useful persistent regimes on the project's free-source proxies, whether they transfer across markets, and whether any resulting strategy improves future outcomes.

Evidence boundaryThe frozen v7 report contains the project's through-2023 proxy evidence. This lesson neither opens nor recalculates its metrics. The toy laboratory proves only its displayed arithmetic.
Modeling limitationEWM half-life is a human-chosen memory shape. Downside threshold zero may not match every investor's required return. Sortino can be unstable when downside is tiny, and standardization can make an extreme but valid observation numerically large. None of these transformations creates causal explanation.

The paper reports a purposeful but parsimonious set, not a theorem that three features are optimal. Adding variables can provide information but also creates more opportunities to overfit. A scientific extension needs a preregistered reason, data-availability audit, causal release timing, selection protocol, and fresh evidence. It should not be justified only because a larger feature set improved a result already inspected.

Calculation: excess return and downside term

Equity earns -1.20% and aligned cash earns +0.01%. Excess return is -1.21%, or -0.0121. Its downside square is 0.00014641 before weighting. The cash subtraction matters even though it is small.

Error diagnosis: ten-day half-life as a ten-day window

A half-life is not a cutoff. A row ten observations old has half today's raw weight; rows older than ten still contribute with progressively smaller weights. Replacing EWM with an equal-weight ten-row average changes the feature.

Error diagnosis: globally scaled features

A scaler fitted on the full sample lets future observations alter earlier standardized coordinates. Fit it inside the permitted trailing 3,000-row model window and retain those parameters for the associated inference.

Advisor explanation

"We transform causally aligned excess returns into DD-10 and Sortino-20/60 using adjusted exponential weights. Each Sortino horizon has its own matching mean and downside denominator. We do not annualize or clip these inputs, and each JM fit standardizes only its trailing 3,000 complete rows."

How to explain this to your advisor

Short version: "The JM does not ingest raw prices. It receives one fast downside-risk summary and two slower reward-to-downside summaries, all derived from excess returns without future data."
