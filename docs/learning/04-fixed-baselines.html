1. Model không đoán chính xác giá ngày mai

Ta giản lược thị trường thành hai state. State chỉ là nhãn toán học; nó không phải sự thật tuyệt đối về kinh tế.

State 0 · equity

Model xem đây là chế độ thuận lợi hơn. Strategy giữ index.

↔
State 1 · cash

Model xem đây là chế độ rủi ro hơn. Strategy giữ local cash proxy.

Ví dụ giả lập

Nếu state là 0, 0, 1, 1, 0, signal equity là 1, 1, 0, 0, 1. Delay và cost được áp dụng sau đó, không nằm trong state model.

3. HMM nhìn return như dấu vết của state ẩn

Hidden Markov Model giả sử ta không nhìn thấy state trực tiếp. Ta chỉ nhìn log return; mỗi state tạo ra một phân phối return khác nhau và có xác suất chuyển state.

state ẩn → phân phối log return → observation ta nhìn thấy
01Lấy 3.000 log returnsChỉ tới ngày đang xét.
02Fit 10 lần10 seed cố định; chỉ nhận fit hội tụ.
03Chọn likelihood cao nhấtKhông chọn bằng OOS performance.
04Viterbi terminal stateChỉ lấy state cuối cửa sổ, rồi lọc causal median.

Project gắn HMM state 0 cho variance thấp và state 1 cho variance cao. Thứ tự này ngăn label tự đổi nghĩa giữa các lần fit.

4. Jump Model gom feature nhưng phạt đổi state

Mỗi ngày là một điểm có ba tọa độ: DD-10, Sortino-20 và Sortino-60 sau standardization trailing 3.000 rows.

min Σ distance(feature_t, center_state_t)² + λ · Σ 1[state_t ≠ state_(t-1)]
Distance

Ngày này giống center nào?

Feature gần center nào hơn thì loss tại state đó nhỏ hơn.

Lambda

Đổi state đắt bao nhiêu?

λ=0 không phạt đổi. Lambda lớn làm path “dính” hơn.

Dynamic programming

Chọn path loss thấp nhất

Online DP chỉ dùng dữ liệu tới terminal day.

Upstream JM sắp state 0 theo cumulative return cao hơn. Model được refit vào tháng 1 và 7; terminal online state được tính mỗi ngày.

HMM state 0 và JM state 0 đều dẫn tới equity, nhưng tiêu chí gắn nhãn khác nhau: variance thấp so với cumulative return cao. Không nên diễn giải hai state là cùng một cấu trúc kinh tế.

Tự kiểm tra

Lambda lớn có luôn tốt hơn không?

Không. Nó giảm switch nhưng có thể giữ sai state quá lâu. Lambda được chọn bằng trailing validation, không bằng cảm giác.

Tại sao HMM phải fit 10 seed?

Likelihood optimization có thể dừng ở nhiều nghiệm cục bộ. Seed cố định giúp thử nhiều điểm đầu mà run vẫn reproducible.

JM MDD tốt hơn B&H ở Hoa Kỳ có đủ pass không?

Không. Gate còn yêu cầu JM Sharpe cao hơn cả HMM và B&H ở cùng market.

Có được làm adaptive để cứu kết quả không?

Không trong protocol hiện tại. Làm vậy sau khi gate fail sẽ đổi model dựa trên OOS đã nhìn thấy.
