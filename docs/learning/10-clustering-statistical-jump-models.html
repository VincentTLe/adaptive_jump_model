Clustering And Statistical Jump Models

How can we group similar market conditions while explicitly charging the model for changing its mind?

Beginner to intermediateJump-penalty laboratoryPaper Equation (1)
The opening question

Should a model move your $100 after every ambiguous day?

You have $100 and a binary allocation rule: state 0 means hold the market; state 1 means hold cash after the required delay. Suppose today's three features look slightly more like a bear pattern than a bull pattern, but tomorrow looks slightly more bullish again. A nearest-pattern rule switches to cash and back. Even before judging returns, that fragile sequence creates two trades and two opportunities for timing error.

A common failure: pointwise labels

0 → 1 → 0

Each day chooses its nearest center independently. Tiny feature movements can create two state jumps around one ambiguous observation.

Persistent path

0 → 0 → 0

A jump model accepts a slightly worse fit on the middle day when avoiding two state changes reduces the full path objective.

A statistical Jump Model, abbreviated JM, begins with clustering. It represents each state by a feature centroid and measures how far each day's standardized feature vector lies from each centroid. It then adds time: every change between consecutive state labels incurs a fixed nonnegative jump penalty.

The penalty is not a brokerage fee and is not subtracted from portfolio wealth. It is regularization inside the model objective. Its purpose is to express a preference for persistent state sequences. Actual transaction costs enter later in validation and backtesting. Keeping these two costs separate prevents a common conceptual error.

The idea to rememberA Jump Model balances two debts. It pays feature-distance loss when a day is assigned to an ill-fitting centroid, and it pays λ whenever the hidden state changes. The optimal path minimizes their sum, so a switch occurs only when better data fit is worth its persistence cost.
Words before notation

Translate clustering into a time-series objective

Feature vector

One row containing the standardized DD-10, Sortino-20, and Sortino-60 values for a date. It is a point in three-dimensional feature space.

Cluster

A collection of feature rows assigned to one representative state. Clustering is unsupervised: no observed bull/bear answer is supplied during fitting.

Centroid

The representative center of one cluster. With squared Euclidean loss, updating a centroid means averaging rows assigned to that state.

Distance loss

Half the squared Euclidean distance from a feature vector to its assigned centroid. Lower means the centroid describes that row better.

Jump penalty

A fixed model-objective cost λ≥0 paid whenever consecutive state labels differ.

Regularization

A preference added to data fit to control model behavior. Here it discourages unnecessary state changes without forbidding them.

Deep explanation: geometry, paths, optimization, and labels

Euclidean distance is ordinary straight-line distance. For three standardized coordinates, squared distance adds the squared coordinate differences. Squaring makes large mismatches matter more and removes signs. The factor one half simplifies some derivations but does not change which path is best if all other objective terms use the declared scale.

Standardized matters because distance depends on units. If DD were numerically one hundred times larger than a Sortino coordinate, it could dominate clustering merely because of representation. Chapter 8 showed that each trailing training window centers and scales its three feature columns before JM fitting.

k-means alternates between assigning each row to its nearest centroid and updating centroids from assigned rows. It ignores order. Shuffling dates leaves the pointwise clustering objective unchanged. The discrete JM becomes k-means when λ=0, because changing states is free.

A state path is the ordered label sequence s0,…,sT−1. The indicator 1{st−1≠st} equals one when a jump occurs and zero otherwise. Summing indicators counts path switches.

Coordinate descent solves one block of variables while holding another block fixed, then alternates. With labels fixed, update centroids. With centroids fixed, find the best ordered labels. The second problem is solved exactly by dynamic programming; Chapter 11 opens that algorithm line by line.

The complete joint problem is still non-convex. Coordinate descent can settle at different local solutions from different initial centroids. The implementation uses multiple initializations and keeps the fit with lowest final objective. This reduces initialization sensitivity but does not prove the global minimum was found.

Anonymous labels have no economic meaning. After fitting, the reference library orders states using training-period returns. With sort_by="cumret", the state with higher cumulative excess return becomes state 0, interpreted as bull/risky, and the other becomes state 1, interpreted as bear/cash. The mapping uses training information, not later strategy results.

Worked example two

Compare a noisy path with a smoother path

Use one invented standardized feature coordinate over eight days: -1.4, -1.1, +0.6, -0.4, +0.8, +1.2, -0.2, and -1.0. Fix state-0 centroid at -1 and state-1 centroid at +1. At λ=0, every day chooses the closer center, producing 0, 0, 1, 0, 1, 1, 0, 0: four switches.

Now set λ=0.5. The optimal path becomes 0, 0, 1, 1, 1, 1, 0, 0. Day 4 at -0.4 fits centroid -1 better by distance, but making that one-day detour would require a switch into state 0 and another switch out. The saved distance loss is not large enough to justify both penalties.

For the smoother path, data loss is 1.50 and two switches cost 1.00, giving total objective 2.50. At a sufficiently large penalty, the optimum becomes all state 0. That path fits several positive observations poorly but pays no transition cost. More persistence is therefore not automatically better; an excessive penalty can erase meaningful changes.

A common failure: treat λ as a probability

The jump penalty can be 5, 70, or 1,200 in the standardized objective. It is not restricted to the interval from zero to one and should not be read as "70% persistent." HMM transition entries are probabilities. JM λ is an objective-scale cost whose behavioral meaning depends on feature scaling, centroid distances, and the data sequence.

Another failure labels the red cluster "bear" because it visually contains a crash in a final chart. Correct state ordering is part of fitting and must use only the associated training returns. Relabeling after viewing out-of-sample performance can reverse signals until a preferred result appears.

Connection to the named paper

Equation (1) makes persistence explicit

Section 3.4 of Shu, Yu, and Mulvey (2024), arXiv:2402.05272v3 defines a two-state statistical JM over the three standardized features from Chapter 8. Equation (1) adds scaled squared Euclidean feature loss and a fixed penalty for every adjacent state change. The authors identify state 0 as bull using higher training-period cumulative excess return and state 1 as bear.

The paper explains that zero penalty reduces the model to k-means, while higher penalties produce fewer transitions and can eventually place all observations in one cluster. Coordinate descent alternates centroid and state-sequence optimization. Ten initializations are run and the lowest objective fit is retained. Section 3.4.1 updates centroids every six months on a rolling 3,000-day window.

Figure 3 visualizes fitted feature centroids by regime for one illustrative fixed penalty. Table 3 compares online state-switch frequency under several fixed penalties. Section 3.4.3 does not declare one universally correct penalty: it selects among candidates monthly using an eight-year online strategy validation. Chapter 12 will separate model fitting from that outer hyperparameter choice.

The paper also discusses continuous and sparse JM variants and possible broader features. They are related literature, not components of Equation (1)'s discrete two-state replication baseline.

Connection to this project

The active baseline delegates fixed JM behavior upstream

src/adaptive_jump/models.py uses the authors' jumpmodels.JumpModel rather than reimplementing fixed JM fitting. For each frozen candidate penalty it passes two states, ten initializations, deterministic seed 0, tolerance and iteration limits, and sort_by="cumret". It rejects non-finite objectives and invalid labels.

The input window contains exactly 3,000 complete rows of DD-10, Sortino-20, Sortino-60, and excess return. A scaler is fitted on those feature rows. The initial eligible fit and scheduled January/July fits estimate centroids separately for each penalty. Between refits, the most recent fitted scaler and centroids remain fixed.

Every day, each candidate model receives the current trailing 3,000 scaled feature rows through upstream predict_online. The adapter validates the returned label vector and retains only its terminal state. Thus semiannual parameter fitting and daily online state inference are different operations.

The frozen grid in research.toml is 0, 5, 15, 35, 70, 150, 300, 600, and 1,200. These are candidates, not values chosen using the current day alone. tests/test_models.py checks refit cadence, causal prefix behavior, cumulative-return ordering, and exact agreement between the adapter's terminal state and upstream online DP.

A separate time-varying DP implementation exists for the preregistered adaptive extension, but no adaptive experiment is opened in this textbook task. Its fixed-penalty nesting test belongs to Chapter 11's algorithm contract, not evidence that adaptation improves performance.

The first sum asks every assigned centroid to explain its feature row. If a row equals its centroid, loss is zero. The second sum counts adjacent label changes and multiplies the count by λ. No penalty is paid at the first row because there is no earlier state inside the sequence.

Holding S fixed, minimizing squared loss sets each centroid to the mean of rows assigned to it. Holding Θ fixed, the best state path can be found recursively because future cost depends on the current ending state, not the complete route used to arrive there. Alternating these two operations cannot increase the objective when implemented correctly, but it may converge to a local solution.

For the Day 4 detour in the toy sequence, switching from state 1 to 0 and immediately back saves some distance loss but adds 2λ. The detour disappears when saved loss is no greater than that added penalty. This local comparison builds intuition; the actual optimizer compares complete paths because one state decision can alter later transition costs.

Evidence and limits

Do not confuse a useful objective with discovered truth

Verified contract

Paper, upstream source, config, adapter, and tests agree on scaled squared distance, off-diagonal fixed penalty, two states, multiple starts, cumulative-return ordering, semiannual refits, and online terminal inference.

Open empirical question

Whether these centroids and persistence choices identify economically useful regimes on each proxy market and remain useful outside the inspected sample.

Evidence boundaryEquation (1) and the cited paper displays establish what the authors specified and reported. The frozen project report separately records proxy behavior. The toy lab establishes neither.
Optimization boundaryCoordinate descent plus restarts is not a proof of global optimality. Euclidean distance favors roughly spherical feature clusters, standardization choices affect geometry, and a fixed penalty expresses one symmetric switching preference.

A state can be statistically compact without being tradable. A persistent path can miss a fast crash or remain defensive during a rebound. Selecting λ by the same sample used for final reporting would overfit persistence. This is why the paper's online validation and the project's sealed boundary gate are part of the model claim, not optional engineering details.

Calculation: objective of a path

If distance losses sum to 7.2, the path changes three times, and λ=5, total objective is 7.2 + 3 × 5 = 22.2. This is a model objective, not portfolio dollars.

Error diagnosis: unscaled distance

A numerically large feature can dominate Euclidean geometry because of units. Fit the scaler only on the trailing training window, transform its rows, then fit centroids.

Error diagnosis: use test returns to name states

That leaks outcomes into the signal. Order anonymous states by cumulative excess return within the associated training fit, then preserve that mapping.

Cumulative recall from Chapters 1-9

Chapter 8 built the JM feature coordinates. Chapter 9's HMM expresses persistence through transition probabilities and emission likelihood. The JM instead uses centroid distance and an explicit fixed switch penalty. Both infer hidden states and require causal endpoint handling.

Advisor explanation

"Our fixed JM minimizes half squared distance to two feature centroids plus lambda for every state jump. Lambda zero nests k-means. We fit multiple starts semiannually, map states by training cumulative excess return, and run upstream online DP daily to keep only the terminal label."

How to explain this to your advisor

Short version: "A Jump Model is temporally regularized clustering. It can tolerate a locally imperfect assignment when that produces a better complete state path."

Important qualification: "Lambda is an objective penalty, not a probability or transaction cost. The fitted path is a model solution under chosen features and geometry, not an observed market regime or a performance claim."
