Data Parity And Proxy Replication

When two files say "S&P 500," are they actually measuring the same investment?

Beginner levelData-definition checklistPaper Section 2
The opening question

Can the label on a file tell you what your $100 earned?

Suppose two researchers each download a file named "Index." Both begin with $100. In the first file, the index level rises from 100 to 108. In the second, the same price change is recorded, but a $2 distribution is also reinvested. The first series reports $108 and an 8% return. The second represents $110 and a 10% total return. The market name is identical, yet the economic investment is not.

File A: price only

$100 → $108

The file follows the quoted index level. It omits the $2 distribution, so the measured return is 8%.

File B: total return

$100 → $110

The distribution is reinvested. The measured total return is 10%, two percentage points above File A.

This difference is not cosmetic. Chapter 2 showed that returns feed wealth. In this project, returns also feed downside deviation, Sortino features, regime fitting, validation Sharpe, positions, and final metrics. Changing the input definition can change every later decision even when all model code is identical. A scientific comparison must therefore begin with the meaning of each column, not with whether the CSV opens successfully.

The idea to rememberA ticker or index name is an address, not a complete data definition. Exact reproduction requires the economically material definition, timing, and coverage to match. When they do not match, use an honest replication label and preserve the difference.
Words before notation

Define the dataset as carefully as the model

Provider

The organization or service that publishes the observation. Bloomberg, Yahoo, FRED, and the Bank of Japan can expose different histories, fields, revisions, and legal terms.

Instrument

The economic object measured: an equity index, Treasury bill, interbank rate, or call rate. Similar maturities do not make different instruments identical.

Field

The selected value inside a source, such as Close, adjusted close, yield, price index, or total-return index level. Field names need source documentation.

Return definition

Whether distributions are excluded, paid out, or reinvested, and whether the value is gross or net of taxes and fees. This determines the wealth represented.

Coverage

The first and last usable dates and all gaps between them. A series beginning in 1988 cannot supply the paper's training history beginning in 1970.

Availability

When an observation could actually be used. Publication lag differs from the date printed on the observation and matters for causal backtests.

Data parity means matching the source study on every outcome-relevant data dimension. Reproduction aims to run the named study with no unresolved material data or protocol difference. Replication asks the same research question while deliberately documenting one or more differences. A proxy is a substitute used because the exact object is unavailable, unaffordable, undocumented, or legally restricted. A proxy can be useful, but its convenience does not make it equivalent.

Provenance is the recorded history of where data came from and how it was retrieved and transformed. A good provenance record includes source ID, URL, query parameters, retrieval time, units, frequency, timezone, field, raw-file hash, canonical-file hash, quality facts, and config identity. Provenance does not guarantee that the provider's definition matches the paper. It lets another researcher identify exactly what was used and investigate the mismatch.

Local currency means each index is measured in its home currency rather than translated into one investor currency. This avoids adding exchange-rate returns to the paper's comparison. Common trading days are dates on which the series being compared all have valid observations. Different market holidays mean calendar dates cannot be aligned by position alone.

Missingness means the source does not provide a usable value for a required observation. It is different from a measured zero. Zero says the quantity was observed and equaled zero; missing says the value is unknown or unavailable. Deleting, filling, or carrying a missing value changes the sample and sometimes the information set. A parity review therefore records both the count and the treatment of missing observations instead of hiding them during CSV cleaning.

Worked example two

A three-month rate is still not a complete cash definition

Imagine the paper says "three-month Treasury Bill Yield." A free database offers a three-month interbank rate. Both are annual percentage quotes and both mention three months, but one concerns government borrowing and the other concerns lending between banks. Their credit, liquidity, market convention, publication frequency, and history can differ. Replacing one with the other changes the cash return and therefore changes excess returns, features, strategy returns, and Sharpe ratios.

For a simple teaching approximation, an annual quote of 3.65% divided by 252 trading days gives about 0.0145% per day. Applied to $100, that is roughly 1.45 cents for one trading day. But this arithmetic does not establish the correct conversion for every bill quote. Discount-basis yields, effective yields, day-count conventions, settlement rules, and compounding conventions are different objects. The source study must disclose enough detail, or the implementation must record its chosen approximation as a deviation.

A common failure: silently joining definitions

Suppose a free price index reaches back to 1970, while a total-return index starts in 1988. It is tempting to place the price history before 1988 and the total-return history after 1988 so the file appears complete. That splice silently changes the meaning of one column on a hidden date. Features before and after the boundary no longer measure the same investment, and a model may learn the artificial definition change. The frozen project forbids definition splicing and shifts the eligible evaluation start instead.

Another failure is to treat a missing observation as zero return or forward-fill an equity index through an unknown gap. A holiday with no new observation is not the same as a verified zero return. A stale monthly cash quote can sometimes be carried under an explicit availability and staleness rule, but that is a protocol decision. The project records those rules separately and rejects values beyond the allowed age.

Timezone and session boundaries matter for the same reason. A date such as 2020-03-16 can refer to closes reached at different moments in Tokyo, Frankfurt, and New York. The paper studies each market independently, so the project does not pretend those closes were simultaneous. For daily single-market returns, consecutive valid local observations are the relevant sequence. A later cross-market strategy would require a separate information-timing contract.

Connection to the named paper

What Section 2 actually fixes

Section 2 of Shu, Yu, and Mulvey (2024), arXiv:2402.05272v3 states that the risky assets are daily total return series for the S&P 500, DAX, and Nikkei 225 from the start of 1970 through the end of 2023. It identifies Bloomberg Terminal as the equity provider and Global Financial Data as the provider of each country's three-month Treasury Bill Yield. Footnote 5 says the indices are denominated in local currencies, so exchange-rate risk is excluded.

Figure 1 shows cumulative excess returns over that period. Table 1 reports annualized variance, covariance, and correlation, with covariance and correlation calculated on common trading days. Section 2 also says the three markets are tested independently and that extreme returns are not processed as outliers. These statements constrain provider, return type, cash instrument, currency, dates, market treatment, calendar alignment, and outlier policy. Matching only the market names is therefore insufficient.

The paper does not publish the exact Bloomberg identifiers, GFD series identifiers, raw snapshots, adjustment metadata, or a complete yield-to-daily-return conversion in the text we verified. Those open details matter for exact data parity. Asking the authors for identifiers or permissible snapshots is not criticism; it is the normal act of separating what the paper specifies from what a reproducer must infer.

Connection to this project

Why the current work is labeled proxy replication

The frozen source contract lives in research.toml. The United States candidate uses Yahoo ^SP500TR and FRED DTB3, but the equity history begins after the requested 1970 start and the paper's exact source identifiers and conversion remain undisclosed. Germany uses Yahoo ^GDAXI and FRED/OECD IR3TIB01DEM156N; the cash series is a monthly interbank proxy rather than a Treasury bill. Japan uses Yahoo ^N225, explicitly a price-index proxy that excludes dividends, plus a monthly Bank of Japan call-rate proxy.

These are definition-first disclosures, not excuses added after viewing a result. The config forbids synthetic backfill, definition splicing, forward-fill, imputation, and outlier removal. It caps all permitted evidence at 31 December 2023. When history is too short, the out-of-sample start is shifted until 3,000 complete feature observations and eight calendar years of online validation are available. That preserves causal history but changes the paper's 1990 start, so the label remains proxy replication.

The manifest proves which bytes and transformations entered a run. It does not prove that Yahoo equals Bloomberg or that an interbank rate equals a Treasury bill. Chapter 6 will formalize this distinction between identity evidence and semantic validity.

Mathematical layer

Exact parity is a conjunction, not an average score

Represent the source study's data definition as a tuple. Each component answers one concrete question. The candidate tuple must match every material component for data parity.

The indicator function I(condition) equals one when its condition is true and zero otherwise. Multiplication encodes the word "and." If all seven comparisons equal one, their product is one. If even one material comparison equals zero, the product is zero. This formula deliberately does not assign a scientific percentage to partial matching.

Return arithmetic then shows why one component can matter. Let P0=100, P1=108, and distribution C=2. Price return is 108/100−1=0.08. Total return is (108+2)/100−1=0.10. The two-percentage-point gap enters every downstream calculation that consumes the return.

A hash comparison, introduced next chapter, can establish that two files contain identical bytes. It cannot establish E=1 unless the semantic tuple is also verified. Byte identity answers "is this the same file?" Data parity answers "does this file measure the same object under the same rules?" Both questions are necessary and neither substitutes for the other.

Evidence and limitations

What the current label does and does not say

Verified evidence

The local paper hash, its Section 2 data description, the frozen source classifications, the no-splicing policy, and the acquisition-manifest behavior can be checked directly.

Still unresolved

Exact Bloomberg and GFD identifiers, source snapshots, some adjustment details, and complete cash conversion conventions are not established by the available paper text.

Empirical statusThe completed v7 study is a shifted-window proxy replication. Its generated English report is reached through the workflow in the README. This chapter does not copy metric values or reinterpret that frozen result.
Limit of inferenceA proxy non-replication does not refute the paper. The difference could arise from data definitions, coverage, protocol interpretation, implementation, model behavior, or a combination. Exact author or licensed data would narrow that attribution problem.

Free data are not automatically poor data, and paid data are not automatically correct. The scientific issue is fitness for the stated question and transparent provenance. A well-documented free proxy may support a useful replication question. It simply cannot support the stronger claim that the original data pipeline was exactly reproduced.

Retrieval practice

Choose the claim that the data can support

Recall

A price index and a total-return index share the same market name and dates. Which label is justified before further evidence?

Choose an answer, then check the explanation.
Calculation: price versus total return

A price rises from $100 to $106 and pays a reinvested $3 distribution. Price return is 6%. Total return is 9% because ($106 + $3) / $100 - 1 = 0.09. Omitting the distribution understates ending wealth by $3 in this one-period example.

Error diagnosis: "the correlation is close, so the data are equivalent"

High correlation describes co-movement, not identity. Two series can correlate closely while differing in distributions, levels, volatility, tail observations, start date, or cash subtraction. Those differences can alter features and strategy metrics.

Cumulative recall from Chapters 1-4

Data definition determines the risky and cash returns from Chapter 2. Those returns create wealth and risk from Chapter 3. Their patterns become regime evidence in Chapter 4. A model cannot repair an input series that measures the wrong investment.

Advisor explanation

Therefore I call the result a proxy replication and preserve every source and hash instead of claiming exact reproduction."

How to explain this to your advisor

Short version: "Data parity is part of the experiment, not a preliminary clerical task. Provider, instrument, field, total-return treatment, currency, calendar, cash rate, and history can all change the result."

Important qualification: "Our free bundle is transparent and reproducible, but it is intentionally labeled proxy replication. That label limits the claim without making the exercise useless."
