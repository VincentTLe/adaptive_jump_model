Rules Learned From This Session

  • Do not use full-sample standardized diagnostics as live model/backtest features.
  • Use train-only, rolling, or expanding normalization in model/backtest code.
  • Offline DP/Viterbi paths are retrospective segmentation; backtests need causal/online states.
  • State labels are arbitrary. State mapping must be train-only and deterministic.
  • Full mode must include walk-forward validation, hyperparameter grids, delay/cost sensitivity, and multiple symbols.
  • If mapping collapses or cannot prove state semantics, mark that model/run invalid instead of silently using identity.
  • Figures must communicate market meaning: cumulative return/equity with shaded regimes, drawdown, turnover, exposure, and clear captions.

Visualization Standard

  • Main figure: cumulative return or equity curve with low-risk/high-risk regime shading on the same timeline.
  • Small multiples: HMM, Fixed JM, Adaptive JM use the same y-axis and window.
  • Use pale shading, clear legend labels, and captions with symbol, sample window, causal/online status, delay, cost, and mapping rule.
  • Separate statistical regime quality from economic performance: path diagnostics plus equity/drawdown/backtest tables.
