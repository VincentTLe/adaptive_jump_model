Part III · Features And Models

Hidden Markov Models From Zero

How can a model reason about a market condition that we cannot observe directly?

Beginner to intermediateViterbi path laboratoryPaper Section 3.3
The opening question

Can the same loss come from two different worlds?

You hold $100 in the market and lose 1%, ending with $99. That observation is compatible with at least two stories. In a calm market whose daily changes are usually small, -1% may be unusually bad. In a turbulent market whose changes are often large, -1% may be ordinary. The $1 loss is visible; the condition that generated it is not.

Observed variable

$100 → $99

The return can be measured. It is an output emitted by an unknown market condition.

A common failure: certainty from one row

-1% ≠ known state

A single return does not reveal its generator. The model weighs how plausible the observation is under each state and how persistent each state is.

A Hidden Markov Model, abbreviated HMM, separates what is seen from what is inferred. The seen sequence is daily return. Behind each return sits a discrete hidden state. In the two-state version used here, one state has lower conditional variance and the other has higher conditional variance. Each state describes a different probability distribution for returns.

The model also represents persistence. If today is in the low-volatility state, tomorrow may be more likely to remain low than to switch high. The exact probabilities are estimated, not declared by the state names. By combining emission plausibility with transition plausibility across a sequence, the model finds a state path that explains the observed returns.

The idea to rememberAn HMM is two linked stories: states move according to transition probabilities, and each hidden state emits observations according to its own distribution. Viterbi decoding finds the single most probable state path under the fitted model; it does not make hidden truth directly observable.
Words before notation

Name the probability machinery before using it

Latent or hidden

Not directly observed. A regime is inferred from returns and model assumptions rather than read from a market data field.

Observation

The measured value at one date. The paper's HMM observation is a daily log total return.

Emission

The probability distribution of an observation conditional on a state. A Gaussian emission has a state-specific mean and variance.

Transition

A move from yesterday's hidden state to today's hidden state. A transition matrix assigns a probability to every permitted move.

Likelihood

How compatible model parameters are with the observed sequence. HMM fitting seeks parameters with high data likelihood.

Viterbi decoding

A dynamic-programming algorithm that finds the most probable complete hidden-state path for fixed HMM parameters.

A random variable can take different values with specified probabilities. A probability distribution describes those possibilities. The Gaussian, or normal, distribution is summarized by a mean and variance. Mean locates its center. Variance measures squared spread, while standard deviation is the square root of variance and has the same unit as return.

Conditional means "given something." An emission probability asks how plausible return rt is given state st. It does not ask how likely the state is before seeing the return. The HMM combines the emission with the probability of arriving from an earlier state.

The Markov property says the distribution of the next hidden state depends on the current hidden state, not the entire earlier state history, once the current state is known. This is a simplifying assumption. It does not say returns are independent through time, and it does not prove markets truly obey a first-order Markov chain.

The transition matrix has one row for the previous state and one column for the next state. Every row sums to one. Diagonal entries describe staying in the same state. Large diagonal probabilities imply persistence. Initial probabilities describe the state before the first observation in a fitted window.

Parameters are quantities estimated during fitting: initial probabilities, transition probabilities, and the means and variances of the state emissions. A local optimum is a fit better than nearby alternatives but not necessarily best among all possible parameters. HMM likelihood is non-convex, so different initializations may end at different solutions.

A restart fits the model again from another deterministic initialization. Convergence means the optimization change has become smaller than a declared tolerance; reaching the iteration cap is not equivalent. The project accepts only strictly converged restarts and then retains the accepted fit with highest log-likelihood.

Label switching means numeric state names are arbitrary. A fitted package may call the volatile state 0 in one run and 1 in another. Economic interpretation requires a deterministic mapping. The project orders estimated conditional variances: lower variance becomes state 0 and higher variance becomes state 1. Equal variances are rejected because the mapping would be ambiguous.

A terminal state is the last decoded state in a window. The paper repeatedly fits or decodes through date t and keeps only that endpoint as the online state for t. A causal majority filter then smooths the sequence of daily terminal states using only current and earlier endpoints.

Worked example two

Combine emissions with persistence across several days

Imagine a teaching HMM with two states. Low volatility has mean +0.05% and standard deviation 0.60%. High volatility has mean -0.30% and standard deviation 1.80%. Both states have a 90% probability of staying and a 10% probability of switching on the next day. These numbers are invented and are not fitted project values.

The six returns are +0.4%, -0.3%, +0.2%, -2.5%, -1.5%, and +0.6%. The first three small movements fit the narrow low-state distribution well. The -2.5% shock is far from the low-state mean but not unusual under the wider high-state distribution. Once a high state is selected, the 90% persistence probability makes remaining high plausible when -1.5% follows.

For the default values, Viterbi decoding chooses low, low, low, high, high, high. It does not classify each day independently. The +0.6% final return might individually look calm, but a switch back to low incurs a low 10% transition probability. The complete path through high can still have greater joint probability.

A common failure: choose the nearest state each day

If each return is assigned only to the Gaussian density with the highest value, transition probabilities disappear. The result can flicker between states on alternating observations. That is a Gaussian mixture classification, not HMM path decoding. The HMM's distinctive structure is the joint tradeoff between fitting each observation and maintaining a plausible state sequence.

Another common failure compares raw probability products over 3,000 days. Multiplying thousands of values below one underflows toward zero in floating-point arithmetic. Implementations work with log probabilities, converting products to sums. This is numerically safer and preserves ordering because the logarithm is monotone.

Lower persistence makes switching cheaper in probability terms, so the path follows individual emissions more readily. Very high persistence demands stronger or repeated evidence before changing state. Persistence can reduce noise but also causes latency: after the world changes, several observations may be needed before a switch becomes the most plausible path.

Connection to the named paper

Section 3.3 turns daily rolling fits into online states

Section 3.3 of Shu, Yu, and Mulvey (2024), arXiv:2402.05272v3 uses a two-state Gaussian HMM on daily log total returns. A 3,000-day training window moves forward daily. Ten initializations address the non-convex likelihood, and the fit with highest log-likelihood is retained. Regimes are distinguished by estimated conditional volatility.

On the window ending at day t, the Viterbi algorithm decodes a state sequence and the authors extract only the last state as the online inferred regime. They identify endpoint instability as a practical problem: the last decoded state can be noisier than interior in-sample labels. The paper therefore applies a median filter, implemented as a trailing rolling mean of terminal labels, and signals high volatility only when that mean exceeds 0.5.

Figure 2 shows rolling conditional return and volatility estimates for the S&P 500. Table 3 compares state-switch frequency under fixed HMM filter lengths and fixed JM jump penalties. Those displays describe the paper's evidence; this chapter does not copy their values. Section 3.4.3 later chooses the HMM filter length monthly through the same online strategy-validation framework used for JM penalty selection.

Connection to this project

The adapter makes every acceptance rule auditable

src/adaptive_jump/models.py sends only equity_log to hmmlearn.GaussianHMM. This differs from the JM's three standardized excess-return features. The adapter requires exactly 3,000 finite returns, uses two components and diagonal covariance, and runs deterministic seeds 0 through 9 from the frozen config.

Each restart must have at least two likelihood-history entries and an absolute final change below tolerance. Failed fits are recorded. Among accepted fits, the adapter selects maximum log-likelihood, orders state variances, rejects ties, maps lower variance to 0 and higher variance to 1, predicts the Viterbi path, and retains the terminal label. Daily diagnostics store date, window bounds, seed, score, ordered variances, accepted-start count, and failures.

The causal filter creates one candidate series per configured window k. For k=0, it leaves terminal states unchanged. Otherwise it computes the trailing mean and returns state 1 only when the mean is strictly greater than 0.5. A tie therefore maps to 0. The v7 config records an expanded frozen grid after its preregistered boundary procedure; Chapter 12 explains why candidate grids are checked before metrics open.

tests/test_models.py checks strict convergence, restart selection, deterministic variance labels, causal filtering, daily rolling windows, terminal extraction, and checkpoint-prefix validity. A checkpoint is accepted only as a contiguous causal prefix on identical dates. These checks prevent silent implementation drift; they do not prove Gaussian emissions or two states are true descriptions of markets.

The first formula factors a complete path. Start in one state, emit the first return, then repeatedly transition and emit. For fixed parameters, Viterbi does not enumerate all 2T+1 two-state paths. At each date and ending state, it retains only the best preceding score and a backpointer. After the final maximum, backpointers reconstruct the optimal path.

Fitting is a separate problem. The hidden states are unknown, so HMM training iteratively raises observed-data likelihood by estimating state responsibilities and updating parameters. Non-convexity explains restarts and explicit convergence checks. Viterbi then decodes using the selected fitted parameters.

For filter length k>0, the paper's causal smoothing can be written s~t=1{1k∑i=t−k+1ts^i>0.5}, where each hatted state is a terminal output that existed on its own date. This filter adds persistence and latency but never uses future terminal states.

Evidence and limits

Know what an inferred state can and cannot mean

Verified implementation facts

The locked paper, frozen config, adapter, and tests align on two Gaussian states, 3,000 rolling log returns, daily fitting, ten restarts, volatility labels, Viterbi terminal extraction, and causal filtering.

Unverified economic interpretation

A high-volatility label is a statistical category under fitted assumptions. It is not proof of recession, crisis causation, or a profitable future trade.

Evidence boundaryThe paper's Figure 2, Table 3, Table 4, and Figure 6 report its empirical HMM behavior. The project report holds separate proxy evidence. This lesson cites locations without reopening or copying result values.
Assumption boundaryTwo states, Gaussian tails, state-conditional distributions, first-order transitions, rolling-window length, and filter choices are modeling assumptions. Financial returns can be skewed, heavy-tailed, structurally changing, and influenced by information outside return history.

Endpoint state uncertainty matters because the investment application needs today's state, exactly where retrospective decoding is least supported by future context. Filtering reduces flicker but delays switches. Multiple restarts reduce sensitivity to initialization but cannot guarantee a global optimum. These are reasons for honest validation and comparison, not reasons to discard HMMs or treat them as ground truth.

Retrieval practice

Separate observation, probability, path, and label

Recall

Why does the project order HMM states by fitted variance?

Choose an answer, then check the explanation.
Calculation: a transition row

If low stays low with probability 0.92, its probability of switching high must be 0.08 in a two-state model because the row sums to one. These probabilities describe state movement, not return size.

Error diagnosis: highest emission each day

Independent daily classification ignores transition probabilities and is not Viterbi decoding. Compare complete path scores using both emissions and transitions.

Error diagnosis: filter centered around today

A centered filter includes future terminal states and leaks information. The valid filter ends at today and looks backward only.

Cumulative recall from Chapters 1-8

Returns describe changes in wealth, risk distinguishes spread from harmful loss, and regimes are latent persistent conditions. Honest data and walk-forward timing constrain every fit. Unlike the Chapter 8 JM features, this HMM consumes rolling daily log returns directly.

Advisor explanation

"Our HMM is a two-state probabilistic baseline. We fit ten deterministic starts daily on trailing log returns, require strict convergence, choose maximum likelihood, label states by conditional variance, keep only the Viterbi endpoint, and smooth those causal endpoints with a trailing majority rule selected later."

How to explain this to your advisor

Short version: "An HMM explains returns through persistent hidden states with different Gaussian distributions. Viterbi combines observation fit and transition probability to produce one most probable path."

Important qualification: "The endpoint label is model output, not observed truth. Daily refitting, convergence checks, deterministic labels, and causal smoothing make it auditable; they do not remove distributional assumptions or latency."
