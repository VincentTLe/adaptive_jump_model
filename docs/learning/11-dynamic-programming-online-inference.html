Dynamic Programming And Online Inference

How can we find the best persistent state path without checking every possible path, and how can we do it without future data?

Intermediate levelBellman table laboratoryPaper Section 3.4.2
The opening question

Must we inspect more than a billion paths before moving $100?

A two-state model observes 30 days before deciding whether your $100 should remain in the market or move toward cash after the required delay. Each day can be state 0 or state 1, so there are 230, or 1,073,741,824, complete paths. Checking every path is possible in theory but wasteful. A 3,000-day research window would have an unimaginable number of possibilities.

A common failure: brute force

Over 1 billion

Enumerating every 30-day binary sequence repeats the same partial calculations and becomes impossible as the window grows.

Dynamic program

2 states per day

For each ending state, retain only the cheapest way to arrive there. Any more expensive route to the same endpoint can never become optimal later.

This elimination rule is the heart of dynamic programming, or DP. At Day 10, suppose two partial paths both end in state 1, but one already costs 7 and the other costs 9. Every future continuation from state 1 adds exactly the same future possibilities to both. The path costing 9 can be discarded permanently.

DP is not an approximation here. For fixed centroids and penalties, it finds an exact minimum-cost state path. The computational gain comes from recognizing that the future needs to know only today's ending state and accumulated best cost, not the complete route by which that state was reached.

The idea to rememberDynamic programming summarizes every relevant past path by one best value per possible current state. Offline traceback reconstructs a full path after seeing the window. Online inference at date t uses only the value row available through t and keeps its cheapest terminal state.
Words before notation

Separate values, pointers, paths, and information sets

Subproblem

A smaller question reused inside the larger problem. Here: the cheapest path through date t that ends in state k.

Value

The minimum accumulated objective for a subproblem. A value is a cost, not a state probability or portfolio return.

Recurrence

A rule that computes today's values from yesterday's values, today's losses, and transition penalties.

Backpointer

The previous state that achieved a minimum. Saving pointers allows an optimal final path to be reconstructed backward.

Terminal state

The cheapest ending state for the currently available prefix. It is the only path element needed for today's online output.

Traceback

Starting from the cheapest final state and following saved backpointers to recover earlier labels.

A loss matrix has one row per date and one column per state. Entry L(t,k) measures the cost of assigning observation t to state k. For the discrete JM, it is half squared distance from the standardized feature vector to centroid k.

A penalty matrix gives the cost of moving from each previous state to each arrival state. The fixed two-state JM has zero on the diagonal and λ off the diagonal. Remaining in the same state is free; changing either direction costs the same amount.

Optimal substructure is the property that an optimal full path contains optimal relevant partial paths. If the cheapest path ending in state 1 at Day 5 used a non-cheapest earlier route ending in the same state, replacing that route with the cheaper one would improve the full path, contradicting optimality.

The Bellman principle is the general idea behind this recurrence: make the current decision using optimal values of smaller preceding problems. The name does not imply a new market assumption. It is an algorithmic structure created by additive losses and state-dependent transition costs.

Offline decoding uses the full chosen window and reconstructs labels for every row. A label in the middle can depend on observations later in that window. Online terminal inference asks only which state minimizes cost at the current endpoint using observations available through that endpoint.

A prefix is the sequence from its start through a current endpoint. If we compute online terminal labels for Day 1, then Days 1-2, then Days 1-3, the earlier emitted labels are not retroactively replaced by a later full-window traceback. Doing so would create hindsight regimes rather than live-sample outputs.

Time complexity describes how operations grow with input size. With T dates and K states, the general recurrence compares K previous states for each of K arrival states on each date: O(TK2). For two states, growth is linear in dates.

Space complexity is stored information. A full value table and backpointers use order TK memory. If only the latest terminal state is needed, values can be updated with two rows, though the upstream implementation returns the complete value matrix for its online sequence.

Worked example two

Fill one Bellman row by hand

Use two states and jump penalty 0.7. On Day 1, losses are 0.1 for state 0 and 2.0 for state 1. There is no incoming transition, so the first value row is simply [0.1, 2.0]. On Day 2, losses are [0.3, 1.5].

To end Day 2 in state 0, either stay from state 0 for cost 0.1 + 0, or jump from state 1 for cost 2.0 + 0.7. The cheaper predecessor costs 0.1. Add today's state-0 loss 0.3, giving value 0.4. The backpointer for this cell is state 0.

To end Day 2 in state 1, either jump from state 0 for 0.1 + 0.7, or stay from state 1 for 2.0 + 0. The first predecessor is cheaper. Add today's state-1 loss 1.5, giving value 2.3. Its backpointer is state 0. Since 0.4 is lower than 2.3, the online terminal state at Day 2 is 0.

Continue through five rows. The online terminal sequence becomes 0, 0, 1, 0, 1. But traceback after the fifth row gives the full-window path 0, 0, 1, 1, 1. Day 4 differs: future Day 5 makes remaining in state 1 globally cheaper for the completed window, while Day 4's live endpoint had favored state 0.

A common failure: publish the offline path as online history

If the final traceback path is plotted as though every middle label existed in real time, Day 4 is rewritten using Day 5. That can make transition timing look cleaner and improve a backtest impossibly. The valid online stream stores the terminal choice made at each endpoint before the next observation arrived.

Another failure uses greedy nearest-centroid labels and calls them DP. Greedy choice keeps only today's smallest loss. DP keeps one accumulated best value for every possible ending state, allowing a locally worse state to survive when it leads to a cheaper persistent path later.

At zero penalty, each value row reduces to the accumulated cheapest pointwise losses and online/offline labels agree with the minimum-loss state on each day. Raising the penalty changes accumulated comparisons. It may cause online labels to lag a new local pattern and lets later evidence alter earlier labels only inside an offline traceback.

Connection to the named paper

Section 3.4.2 uses only the last state

Section 3.4 of Shu, Yu, and Mulvey (2024), arXiv:2402.05272v3 states that DP solves the discrete state-sequence block in linear time when model parameters are fixed. Section 3.4.2 then defines the live-sample use: with the most recently fitted centroids and a fixed candidate penalty, process the trailing 3,000 feature rows available through day t and extract the last state.

A nearest-centroid endpoint would ignore temporal information. DP carries the path's persistence cost into the current terminal comparison. The paper emphasizes that tomorrow's feature vector is unavailable. Its Figure 4 contrasts a later full-window in-sample regime path with daily online inferred regimes around the 2020 episode, illustrating that online detection can lag turning points.

Latency is not the trading delay. Model latency is the observations needed before changing the inferred state becomes worthwhile. Trading delay is the later operational gap between an available signal and the return first controlled by its position. A persistent DP path may have latency even when execution is immediate.

Connection to this project

Reference behavior is tested, not assumed

The fixed baseline in src/adaptive_jump/models.py calls upstream JumpModel.predict_online on each scaled trailing window. The upstream library builds the half-squared-distance loss matrix, computes all DP value rows, and takes each row's minimum-cost state. The adapter validates shape and labels, then stores only the final label for the current date.

src/adaptive_jump/tv_jump.py contains the single custom time-varying-penalty DP reserved for the preregistered extension. Its recurrence uses penalty_seq[t] for the transition arriving on date t; index zero is unused because the first state has no incoming jump. A scalar sequence is expanded to matrices with zero diagonals and date-specific off-diagonal penalties.

values[0] = loss[0]
for t in 1..T-1:
    values[t, k] = loss[t, k] + min_j(
        values[t-1, j] + penalty[t, j, k]
    )

tests/test_tv_jump.py compares tiny random DP problems with brute-force enumeration, including two and three states. It verifies the returned path actually achieves the optimum. Zero penalty must equal pointwise row minima; a huge penalty must forbid switching.

Most importantly, a constant penalty sequence must match upstream fixed DP exactly: objective value, full assignment, and online value matrix. The fitted time-varying class with a constant sequence must also match upstream fitted labels, objective, and online labels under identical starts. This is a numerical nesting contract. It says the extension contains the fixed baseline; it does not say a changing penalty improves scientific outcomes.

The scalar penalty-sequence constructor rejects non-finite, negative, or non-vector inputs, and the solver checks matrix shape. Adaptive experiments, parameter selection, and any performance claim remain outside the current textbook task and behind the replication gate.

Mathematical layer

Derive the recurrence from the last transition

V0(k)=L(0,k)Vt(k)=L(t,k)+minj[Vt−1(j)+Λt(j,k)]s^t=arg minkVt(k)
SymbolMeaningUnit or convention
tCurrent row index0 through T−1
jCandidate previous state0 through K−1
kCandidate arrival state0 through K−1
L(t,k)Observation loss for assigning row t to state kNonnegative objective cost
Λt(j,k)Cost of transition j→k arriving at tZero to stay in fixed JM
Vt(k)Minimum prefix objective ending in state kAccumulated objective cost

Every path ending in state k at date t must come from exactly one previous state j. Its cost is a best prefix ending at j, plus the transition into k, plus today's observation loss. Taking the minimum over j therefore covers every relevant path without listing them.

For fixed JM, Λt(j,k)=λ1{j≠k} for every arrival date. A date-varying scalar replaces λ by λt. If all λt equal the same value, every recurrence cell equals the fixed reference, which explains the nesting test.

The online terminal equation needs only values through t. Offline traceback additionally stores Bt(k), the minimizing predecessor, starts from the final minimum, and follows pointers backward. Both are exact for their different questions.

Evidence and limits

An exact algorithm can still optimize the wrong model

What exactness means

For fixed losses and penalties, DP returns a globally minimum state sequence. Oracle tests verify the implementation against enumeration and the upstream reference.

What exactness does not mean

Centroids, features, penalty choice, data definitions, state interpretation, or future economic usefulness are not proven correct by solving their objective exactly.

Evidence boundaryThe tests establish numerical contracts on deterministic inputs. The paper supplies empirical examples of online latency. Neither source turns a hidden state into observed truth.
Online limitationThe endpoint has less future context than an interior offline label. Persistence prevents flicker but creates detection latency. A trailing 3,000-row window also drops old information as it advances, so consecutive online problems are related but not identical prefixes forever.

Computational efficiency makes repeated evaluation possible; it does not justify searching unlimited model variants. Every extra penalty rule or asymmetry creates another scientific choice. The current extension preserves a scalar time-varying penalty and an exact fixed baseline, but adaptive performance remains unopened until the earlier replication gate and research freeze permit it.

Calculation: one Bellman cell

Previous values are [1.2, 2.0], λ=0.5, and today's state-1 loss is 0.3. Arriving in state 1 costs min(1.2 + 0.5, 2.0 + 0) + 0.3 = 2.0.

Error diagnosis: penalty on the first row

There is no incoming state inside the window at row zero, so initialize values from row-zero losses only. Charging a jump there changes the objective.

Error diagnosis: future-revised regime chart

A full-window traceback can relabel interior dates. For a live backtest, save the terminal state produced at each historical endpoint and never replace it using later rows.

Advisor explanation

"For each day and possible ending state, DP stores the cheapest prefix cost and predecessor. That reduces exponential path search to order T times K squared. Our online signal uses only the current terminal argmin; full traceback is an offline explanation and may revise interior labels."

How to explain this to your advisor

Short version: "DP is exact because paths ending in the same current state share all future options, so only the cheapest such prefix must survive."

Important qualification: "Exact objective minimization is an algorithm claim. It does not validate the objective economically, and an offline traceback must never be substituted for historical online terminal states."
